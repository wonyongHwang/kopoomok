파일별 역할 상세 설명

오목 게임 소스코드의 각 파일별 핵심 기능과 상호작용 흐름

index.html

사용자 인터페이스와 게임 루프를 관리하는 핵심 파일

보드, 버튼, 좌표 레이블, 점수, 진행률, 트리 시각화 UI 구성
사용자 돌 두기 → 유효성 검사 → 착수 → 승리 체크 → triggerAIMove() 호출
난이도 버튼(Easy/Medium/Hard)으로 탐색 깊이 조절
ai.js

인공지능의 핵심 로직을 담당하는 파일

getPossibleMoves(): 주변 빈칸만 후보 수 생성, 3×3 금수 필터 적용
evaluateBoard(): 가로/세로/대각 6칸 윈도우 패턴 가중치로 평가
minimax(): 미니맥스 알고리즘 + 알파-베타 가지치기 구현
getAIMove(): 진행률/트리 데이터 생성/히트맵 표시 기능
checkWin(): 승리 조건 판정 로직
threebythree.js

3×3 금수 판정을 위한 유틸리티 파일

정규식과 패턴 매칭을 활용한 금수 판정
흑돌의 쌍삼, 쌍사, 장목 등 렌주룰 검사
visualizer.js

미니맥스 알고리즘의 동작을 시각화하는 파일

minimax가 생성한 트리 구조 시각화(알파, 베타, 점수, 가지치기, 착수 위치)
<ul> 태그 구조로 트리 계층 렌더링
가지치기(pruning) 발생 지점 강조 표시
파일 간 상호작용 흐름
index.html
사용자 착수 → 유효성 검사
threebythree.js
금수 판정 결과 반환
ai.js
최적의 수와 트리 정보 반환
visualizer.js
트리 시각화 데이터 전달
index.html (UI 업데이트)
주요 호출 흐름
  1. 사용자가 돌을 두면 index.html에서 게임 상태 업데이트
  2. AI 턴이면 triggerAIMove() → getAIMove(board, searchDepth) 호출
  3. getAIMove 내부에서 각 후보 착수에 대해 minimax 호출
  4. 최적 수 선택 후 트리 정보가 visualizer.js로 전달되어 시각화
  5. threebythree.js는 유효성 검사 단계에서 금수 판정에 활용
파일 간의 모듈화와 책임 분리를 통해 코드 유지보수성과 확장성이 향상됩니다.
