오목 AI의 알고리즘 구현 상세 분석

ai.js 파일의 주요 함수
  • minimax(board, depth, alpha, beta, player, latestRow, latestCol) - 미니맥스 알고리즘과 알파-베타 가지치기의 핵심 구현
  • evaluateBoard(grid, latestRow, latestCol) - 현재 보드 상태의 점수 계산
  • isWinningMove(grid, who, row, col) - 승리 조건(5목) 판별
  • hasNeighbor(board, row, col) - 주변에 돌이 있는지 검사하여 탐색 공간 최적화
  • lineCheck(grid, who, sRow, sCol, mRow, mCol) - 특정 방향으로 연속된 돌 확인
AI의 핵심 설계 원리

오목 AI는 다음과 같은 주요 설계 원칙을 따릅니다:

  • 깊이 제한된 미니맥스 탐색으로 계산 시간 제어
  • 알파-베타 가지치기를 통한 탐색 최적화
  • 인접 돌 중심 탐색으로 탐색 공간 축소
  • 승리 조건과 점수 계산의 효율적 설계
평가 함수의 특성
  • evaluatePlayerBoard(): 각 플레이어 입장에서의 점수 계산
  • 승리 조건(5목) 달성 시 최대 점수 1,000,000 반환
  • 연결된 돌과 주변 돌 패턴에 기반한 점수 계산
  • 컴퓨터(maximizer)와 사용자(minimizer) 점수 차이를 최종 평가 값으로 사용
minimax()
• 재귀적으로 최적의 수 탐색
• alpha, beta 값으로 불필요한 가지 제거
• 최대 깊이에 도달하면 evaluateBoard 호출
↓
evaluateBoard()
• 현재 보드 상태 평가
• AI(COMP)와 사람(HUMAN)의 점수 차이 계산
↓
isWinningMove()
• 특정 위치에 돌을 놓았을 때 승리 여부 판단
• 가로, 세로, 대각선 4방향 검사
↓
hasNeighbor()
• 특정 위치 주변 8방향에 돌이 있는지 확인
• 탐색 공간을 크게 줄이는 최적화 역할
function minimax(board, depth, alpha, beta, player, latestRow, latestCol) { if (depth === 0) { const val = evaluateBoard(board, latestRow, latestCol); return [val, -1]; // [평가값, 최적 위치] } const opponent = player === COMP ? HUMAN : COMP; if (isWinningMove(board, opponent, latestRow, latestCol)) { return player === COMP ? [1000000, -1] // AI 승리 : [-1000000, -1]; // 사람 승리 } // 최적 위치 탐색 및 알파-베타 가지치기 로직... }
