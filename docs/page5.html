난이도 시스템과 탐색 깊이

AI의 탐색 깊이를 조절하는 난이도 시스템 구현과 영향 분석

난이도 시스템 구현
  • 사용자는 UI에서 Easy, Medium, Hard 버튼을 통해 AI의 난이도를 선택할 수 있습니다.
  • 각 난이도 버튼은 data-depth 속성으로 탐색 깊이(1, 2, 3)를 저장합니다.
  • 선택된 난이도에 따라 전역 변수 searchDepth가 업데이트되어 AI에 전달됩니다.
// index.html 내부 난이도 선택 버튼 <div id="difficulty-selection"> <button class="difficulty-btn" data-depth="1">Easy</button> <button class="difficulty-btn active" data-depth="2">Medium</button> <button class="difficulty-btn" data-depth="3">Hard</button> </div> // 전역 변수와 이벤트 리스너 let searchDepth = 2; // 기본 Medium difficultySelectionDiv.addEventListener("click", (event) => { if (event.target.classList.contains("difficulty-btn")) { searchDepth = parseInt(event.target.dataset.depth); // ... active 클래스 갱신 } });
난이도가 미니맥스에 미치는 영향
  • 난이도는 미니맥스 알고리즘의 초기 탐색 깊이(depth)를 결정합니다.
  • 사용자가 수를 두면 triggerAIMove() 함수가 호출되고 searchDepth 값이 AI에 전달됩니다.
  • getAIMove(board, searchDepth)는 선택된 난이도에 따라 각 후보 수에 대해 미니맥스를 호출합니다.
ply(플라이)란?

체스나 바둑 같은 게임에서 한 명의 플레이어가 한 번 두는 것을 의미합니다.

  • Easy (depth=1): AI는 자신의 한 수만 고려 (1 ply)
  • Medium (depth=2): AI 한 수 → 상대 한 수까지 고려 (2 ply)
  • Hard (depth=3): AI 한 수 → 상대 한 수 → AI 한 수까지 고려 (3 ply)
