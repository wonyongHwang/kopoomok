알고리즘 핵심 구조
  • 미니맥스 알고리즘: 컴퓨터(최대화)와 사용자(최소화) 차례를 번갈아가며 최적의 수를 탐색
  • 재귀 호출: 지정된 깊이까지 게임 트리를 탐색하며 평가 값을 계산
  • 반환 값: [평가 점수, 최적의 수 위치] 형태의 배열 반환
알파-베타 가지치기 구현
  • 알파값: 최대화 플레이어(컴퓨터)가 보장받는 최소 점수
  • 베타값: 최소화 플레이어(사람)가 보장받는 최대 점수
  • 조건 beta <= alpha일 때 탐색을 중단하여 불필요한 노드 탐색을 방지
  • 가지치기를 통해 탐색 시간을 대폭 단축하면서도 동일한 결과 보장
탐색 공간 최적화
  • 이웃 돌 검사: hasNeighbor() 함수를 통해 주변에 돌이 있는 위치만 탐색
  • 승리 조건 우선 확인: 이미 승리한 상황이면 즉시 높은 점수 반환
  • 실제 오목 게임에서는 전체 보드(15x15) 중 유의미한 위치만 검사하여 효율성 향상
