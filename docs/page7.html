전체 호출 흐름도 및 구조도

사용자 착수부터 AI 응답까지의 완전한 흐름과 함수 간 상호작용

호출 흐름 개요

오목 게임에서 사용자가 착수한 후 AI가 응답하기까지의 전체 프로세스를 설명합니다. 난이도 설정에 따라 AI의 탐색 깊이가 결정됩니다.

핵심 함수 호출 순서
  • 사용자 착수 → 유효성 검사 → 착수 처리
  • triggerAIMove() → 현재 난이도에 따른 searchDepth 전달
  • getAIMove(board, depth) → 후보 수 생성 및 평가
  • minimax(board, depth, α, β, maximizing) → 최적 수 탐색
  • evaluateBoard() → 보드 상태 평가
  • 결과 반환 및 UI 업데이트 → 트리 시각화
난이도 설정의 영향
  • Easy (depth=1): 1수 앞을 내다봄
  • Medium (depth=2): 2수 앞을 내다봄 (기본값)
  • Hard (depth=3): 3수 앞을 내다봄

난이도 버튼 → searchDepth → getAIMove → minimax 순으로 전달됩니다.
